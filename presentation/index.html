So how do we actually contribute to GitHub repos?
We have to push and pull. In git, we say that each repository has two copies. We call these local and remote. Every contributor has their own local copy, and remote is treated as the master.
Pushing: sending commits from your local repo to remote. You make changes, and you send them to the main repo for everyone else to see.
Pulling: retrieving new commits from remote. It is a form of synchronisation. You're making sure that local and remote are in the same state. Note, draw a diagram on the board at this point, showing that each user has a local.
